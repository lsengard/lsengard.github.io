说在前面

Gulp是基于Node.js的自动任务运行器，它能自动化地完成js/sass/less/html/image/css等文件的的测试、检查、合并、压缩、格式化、浏览器自动刷新、部署文件生成，并监听文件在改动后重复指定的这些步骤。在实现上，它借鉴了Unix操作系统的管道（pipe）思想，前一级的输出，直接变成后一级的输入，使得在操作上非常简单。通过这篇文章，我们将使用Gulp来改变开发流程，从而使开发更加快速高效。

一、安装gulp

在此之前我们需要Node.js环境，查看安装方法请查阅。

首先我们进行全局安装，这样是为了通过执行gulp命令来进行任务。
我们通过npm来进行安装npm install gulp -g。稍等片刻，等待下载完成你可以用gulp -v来测试是否安装成功。如果在控制台看到了版本号，就证明你安装成功；

然后我们执行npm install gulp --save-dev进行本地安装。这里就有个问题：全局安装了，本地也安装了，这是为什么？那是因为全局安装为了方便执行gulp任务，本地安装是方便调用gulp插件的功能。

这里我们用到了一个gulp的插件gulp-less，就是这个插件来帮我们完成less编译工作。
我们同样用npm安装它，npm install gulp-less --save-dev。
