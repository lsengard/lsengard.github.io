git 简易指南

git 是一款免费的、开源的、分布式的版本控制系统。旨在快速高效地处理无论规模大小的任何软件工程。
每一个 git克隆 都是一个完整的文件库，含有全部历史记录和修订追踪能力，不依赖于网络连接或中心服务器。其最大特色就是“分支”及“合并”操作非常快速、简便。

最近接触git比较多，所以写个博文，以免自己忘记，为了温故而知新。

安装

首先是安装，git官网上提供各种不同平台版本的下载，下载地址请移步这里传送门。
新建仓库
下载安装完成之后，创建一个新的文件夹，打开文件夹，然后执行
git init
以用来创建新的git仓库

克隆到本地
可以执行以下命令创建一个本地仓库的克隆版：
git clone <版本库的网址>
比如，克隆jQuery的版本库。
git clone https://github,/.com/jquery/jquery.git
该命令会在本地主机生成一个目录，与远程主机的版本库同名。如果要指定不同的目录名，可以将目录名作为git clone命令的第二个参数
git clone <版本库的网址> <本地目录名>

工作流
本地仓库由git维护的三棵”树”组成。第一个是我们的工作目录(Working Dir)，是我们持有的实际文件；第二个是缓存区(Index)，这个像是一个缓存区域，临时保存我们的改动；最后是HEAD，指向我们最近一次提交后的结果。用下面这张图更好表述：

添加与提交
我们可以计划我们的改动(把他们添加到缓存区)，可以使用如下命令，
git add <FileName> git add *
这是git基本工作流程的第一步；使用如下命令用来实际提交改动
git commit -m '代码提交信息(一些注释：比如本次提交是什么类型的修改)'
现在，我们的改动已经提交到了HEAD，但是还没有提交到我们的远端仓库中。

推送改动
通过刚才的操作，我们的改动现在已经在本地仓库的HEAD中了。执行如下命令把这些改动提交到远端仓库中：
git push origin master
可以把master换成我们想要的推送的任何分支。
如果我们还没有克隆现有仓库，并想讲我的仓库连接到某个远程服务器，我们可以使用如下命令添加：
git remote add origin <server>
这样我们就能把我们的改动推送到所添加的服务器上去了。

分支
分支是用来将特性开发绝缘开来的。在你创建仓库的时候，master 是“默认的”。在其他分支上进行开发，完成后再将它们合并到主分支上。
分支能更好的让我们在开发中做很多想做的事情，它就像是主分支的一个影子。
例如：创建一个叫做“dev”的分支，并切换过去
git checkout -b dev
删除分支”dev”：
git branch -d dev
除非我们将分支推送到远端仓库，不然该分支就是 他人不可见的：
git push origin <branch>

更新与合并
如果需要更新本地的仓库，可以执行
git pull
以在我们的工作目录中获取并合并远端的改动。
要合并其他分支到当前分支(例如：master)，可以执行
git merge <branch>
两种情况下，git 都会尝试去自动合并改动。但是，自动合并不是每次都能成功，并可能导致冲突(产生冲突的原因有，两个人同时修改了同一个文件的同一个部分)。这时候就需要你修改这些文件来人肉合并这些 冲突了。改完之后，你需要执行如下命令以将它们标记为合并成功：
git add <filename>
在合并改动之前，也可以使用如下命令查看：
git diff <source_branch> <target_branch>

标签
我们在软件发布时候时常会有个版本号，可以执行如下代码添加一个1.0.0的标签
git tag 1.0.0 f6bc925c37
后面那串字符是我们想要标记的提交ID的前10个字符。使用如下命令获取提交ID：
git log --stat
ps:你也可以用该提交 ID 的少一些的前几位，只要它是唯一的。

替换本地改动
假如本地版本有什么奇怪的错误了，你可以使用如下命令替换掉本地版本
git chechout -- <filename>
此命令会使用 HEAD 中的最新内容替换掉你的工作目录中的文件。已添加到缓存区的改动，以及新文件，都不受影响。
假如你想要丢弃你所有的本地改动与提交，可以到服务器上获取最新的版本并将你本地主分支指向到它：
git fetch origin git reset --hard origin/master

总结

目前为止大部分的参考来自这里。git是一个不错的工具，同时这种工作方式也提供了很多新的思路，高效便捷。
